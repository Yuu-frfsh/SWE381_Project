# Product

## Register

product

## Users

Two roles share the app:
- **Stadium Owners**: manage their stadiums, add time slots, view bookings, message organizers
- **Match Organizers**: search stadiums by location/date, reserve slots, message owners

Context: Both are non-technical users doing a practical task. Speed and clarity matter more than visual impressiveness.

## Product Purpose

SoccerHub lets stadium owners list their facilities with time slots, and match organizers find and book those slots. Success means completing a reservation in under a minute with no confusion.

## Brand Personality

Friendly, Simple, Clear. The interface should feel like a well-organized form, not a sports product or a SaaS dashboard.

## Anti-references

- Flashy SaaS dashboards: gradient heroes, big KPI hero metrics, purple/indigo palettes
- Over-designed sports apps: heavy imagery, neon accents, ESPN-style visual noise
- Generic Bootstrap/Material defaults: flat blue buttons, gray everything

## Design Principles

1. **Invisible tool**: the UI should disappear — users complete their task without noticing the design
2. **Green is accent, not identity**: one purposeful green for action; neutrals carry everything else
3. **Clarity over decoration**: if an element doesn't help the user understand or act, remove it
4. **Consistent rhythm**: same spacing rules everywhere so the eye can predict

## Accessibility & Inclusion

WCAG AA minimum. Sufficient color contrast on green buttons. All interactive elements keyboard-reachable.
